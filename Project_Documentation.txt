توثيق المشروع

المقدمة:

يهدف هذا المشروع إلى تصميم وتطوير نظام موزع بسيط لآلة حاسبة موزعة. يقوم العميل بإرسال تعبيرات رياضية (مثل 2+3 أو 5*6) إلى الخادم، الذي يقوم بحساب النتيجة وإرجاعها إلى العميل. كما يعتمد العميل على JavaFX لواجهة المستخدم الرسومية، ويستخدم الخادم Java Network Programming لمعالجة الطلبات وحساب النتائج.

توضح هذه الوثيقة تفاصيل المشروع، من المفاهيم الأساسية إلى التصميم والتنفيذ، مع شرح كل مكون من مكونات النظام.

الفصول:

1. أهداف المشروع
يهدف المشروع إلى:
- استكشاف مفاهيم البرمجة الموزعة عبر تطبيق آلة حاسبة بسيطة.
- تعلم تصميم نظام موزع باستخدام نموذج العميل-الخادم.
- بناء واجهة مستخدم رسومية باستخدام JavaFX.
- التعامل مع طلبات متزامنة من العملاء.
- التعامل مع الأحداث والأخطاء بطريقة مناسبة لضمان استقرار النظام.

2. التقنيات المستخدمة
- لغة البرمجة: Java
- إطار العمل للواجهة: JavaFX
- تقنيات الشبكة: TCP/IP Sockets
- البيئة التطويرية: Java Development Kit (JDK) وبيئة تطوير متكاملة مثل IntelliJ IDEA أو Eclipse.

3. وصف نظام العميل (Client)
واجهة المستخدم:
- حقل لإدخال التعبير الرياضي.
- زر لإرسال التعبير إلى الخادم.
- منطقة لعرض النتيجة أو رسائل الخطأ.

آلية الإرسال:
- يتصل العميل بالخادم عبر Socket.
- يرسل التعبير الرياضي كنص.
- يعرض النتيجة المستلمة من الخادم.

التعامل مع الأخطاء:
- التحقق من صحة الإدخال قبل الإرسال.
- إظهار رسائل مناسبة في حالة أخطاء الشبكة أو التعبير.

4. وصف نظام الخادم (Server)
- يستمع الخادم على منفذ محدد لاستقبال طلبات العملاء.
- لكل اتصال عميل، ينشئ Thread مستقل للتعامل مع الطلبات.
- يستقبل التعبير الرياضي من العميل، يقوم بتحليله وحسابه بأمان.
- يرسل النتيجة أو رسالة خطأ إذا كان التعبير غير صالح.

التعامل مع التزامن:
- دعم عدة عملاء متصلين في نفس الوقت بواسطة Threads متعددة.

التعامل مع الأخطاء:
- معالجة أخطاء التحليل الحسابي.
- التعامل مع انقطاع الاتصال بطريقة مناسبة.

5. بنية النظام
- يعتمد المشروع على نموذج العميل-الخادم.
- كل عميل يتصل بالخادم عبر TCP Socket.
- الخادم يعالج الطلبات ويعيد النتائج بشكل متزامن.

6. تفاصيل التنفيذ
- شرح مفصل لكود العميل، موضح كيفية إنشاء واجهة JavaFX وإرسال التعبير واستقبال النتيجة.
- شرح مفصل لكود الخادم، مع استخدام ServerSocket وThread لمعالجة الطلبات وحساب النتائج بأمان.

7. التعليمات التشغيلية
- خطوات تشغيل الخادم.
- خطوات تشغيل العميل.
- شرح واجهة المستخدم وتعاملها مع الإدخال والنتائج.

8. التحديات والمشكلات
- تأمين تحليل التعبيرات لتجنب الأخطاء والثغرات.
- إدارة التزامن وعدم حجب الخادم.

9. التحسينات المحتملة
- دعم تعبيرات رياضية أكثر تعقيداً.
- إضافة واجهة ويب إضافية.
- تخزين تاريخ العمليات والنتائج.

10. الشاشات والنتائج
- شاشة العميل مع حقل الإدخال وزر الإرسال ومنطقة عرض النتائج.
- شاشة الخادم مع سجل الطلبات والنتائج.

الخاتمة:
يقدم المشروع تجربة تعليمية في بناء نظم موزعة بسيطة، تجمع بين البرمجة الشبكية والمعالجة الحسابية، مع التركيز على استقرار وأمان النظام.

Project Documentation

Introduction:

This project aims to design and develop a simple distributed system for a distributed calculator. The client sends mathematical expressions (such as 2+3 or 5*6) to the server, which calculates the result and returns it to the client. The client uses JavaFX for the graphical user interface, and the server uses Java Network Programming to process requests and compute results.

This document clarifies the project details from basic concepts through design and implementation, explaining each system component.

Chapters:

1. Project Objectives
The project aims to:
- Explore distributed programming concepts through a simple calculator application.
- Learn to design a distributed system using the client-server model.
- Build a graphical user interface using JavaFX.
- Handle concurrent client requests.
- Appropriately manage events and errors to ensure system stability.

2. Technologies Used
- Programming Language: Java
- UI Framework: JavaFX
- Network Technologies: TCP/IP Sockets
- Development Environment: Java Development Kit (JDK) and an IDE such as IntelliJ IDEA or Eclipse.

3. Client System Description
User Interface:
- A field to enter mathematical expressions.
- A button to send the expression to the server.
- An area to display the result or error messages.

Sending Mechanism:
- The client connects to the server via a Socket.
- Sends the mathematical expression as text.
- Displays the result received from the server.

Error Handling:
- Validates input before sending.
- Shows appropriate messages for network or expression errors.

4. Server System Description
- The server listens on a specified port for client requests.
- For each client connection, creates an independent Thread to manage requests.
- Receives mathematical expressions from clients, parses and safely computes results.
- Sends the result or an error message if the expression is invalid.

Concurrency Handling:
- Supports multiple clients concurrently using multiple Threads.

Error Handling:
- Handles parsing and computation errors.
- Appropriately manages disconnections.

5. System Architecture
- The project is based on the client-server model.
- Each client connects to the server via a TCP Socket.
- The server processes requests and returns results concurrently.

6. Implementation Details
- Detailed explanation of client code showing JavaFX UI creation, sending expressions, and receiving results.
- Detailed explanation of server code using ServerSocket and Thread for safe expression evaluation.

7. Operational Instructions
- Steps to start the server.
- Steps to start the client.
- Explanation of user interface interactions.

8. Challenges and Issues
- Securing expression evaluation to avoid errors and vulnerabilities.
- Managing concurrency without blocking the server.

9. Potential Improvements
- Supporting more complex mathematical expressions.
- Adding a web-based interface.
- Storing history of operations and results.

10. Screenshots and Results
- Client screen with input field, send button, and result display area.
- Server screen with request and result logs.

Conclusion:
This project provides an educational experience in building simple distributed systems combining networking and computation, focusing on system stability and security.
