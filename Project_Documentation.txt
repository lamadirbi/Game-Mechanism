وثائق مشروع نظام قائمة مهام موزعة بسيط (محاكاة)

المقدمة:

يهدف هذا المشروع إلى تصميم وتطوير نظام قائمة مهام موزعة بسيط مع واجهة مستخدم رسومية باستخدام JavaFX. يسمح النظام للمستخدم بإدارة مهمات عبر قائمتين منفصلتين، مع إمكانية إضافة المهام وحذفها ومزامنتها بين القائمتين بمحاكاة لعملية التوزيع.

This project aims to design and develop a simple distributed task list system with a graphical user interface using JavaFX. The system allows the user to manage tasks across two separate lists, with the ability to add, delete, and synchronize tasks between the two lists simulating distribution.

توضح هذه الوثيقة تفاصيل المشروع، بدءًا من أهدافه ومكونات النظام، إلى التصميم والتنفيذ، مع شرح مفصل لواجهة المستخدم وعملية مزامنة المهام.

This document details the project from its objectives and system components to design and implementation, including a detailed explanation of the user interface and task synchronization process.

الفصول:

Chapters:

1. أهداف المشروع
يهدف المشروع إلى:
- تطوير نظام بسيط لإدارة المهام يتيح التعامل مع قائمتين للمهام.
- بناء واجهة مستخدم رسومية واضحة وسهلة الاستخدام باستخدام JavaFX.
- تعزيز فهم مفاهيم التوزيع من خلال محاكاة مزامنة المهام بين قائمتين.

1. Project Objectives
The project aims to:
- Develop a simple task management system that allows handling two separate task lists.
- Build a clear and user-friendly graphical interface using JavaFX.
- Enhance understanding of distributed concepts by simulating task synchronization between the lists.

2. التقنيات المستخدمة
- لغة البرمجة: Java
- إطار العمل للواجهة: JavaFX
- البيئة التطويرية: Java Development Kit (JDK) وبيئة تطوير متكاملة مثل IntelliJ IDEA أو Eclipse.

2. Technologies Used
- Programming Language: Java
- UI Framework: JavaFX
- Development Environment: Java Development Kit (JDK) and an IDE such as IntelliJ IDEA or Eclipse.

3. وصف نظام القائمة
واجهة المستخدم:
- تحتوي الواجهة على قائمتين مستقلة للمهام ليعرض كل منهما مجموعة المهام الخاصة به.
- حقل إدخال نص لإضافة مهمة جديدة.
- أزرار لإضافة مهمة، حذف مهمة من كل قائمة، ومزامنة المهام بين القائمتين.

3. System Description
User Interface:
- The interface contains two independent task lists, each displaying its own set of tasks.
- A text input field to add a new task.
- Buttons to add a task, delete a task from either list, and synchronize tasks between the lists.

آلية عمل التطبيق:
- يمكن إدخال مهمة جديدة في مربع الإدخال وإضافتها إلى القائمة الأولى.
- يمكن حذف مهمة مختارة من أي من القائمتين.
- زر المزامنة يقوم بنسخ جميع المهام من القائمة الأولى إلى الثانية كمحاكاة لعملية موزعة.

Application Mechanism:
- New tasks can be input through the text field and added to the first list.
- Selected tasks can be deleted from either of the two lists.
- The sync button copies all tasks from the first list to the second, simulating a distributed synchronization.

4. بنية النظام
- تطبيق مستقل يعمل بواجهة مستخدم رسومية باستخدام JavaFX.
- لا يعتمد التطبيق على خوادم خارجية أو اتصالات شبكية فعلية.

4. System Architecture
- Standalone application with a graphical user interface built using JavaFX.
- The application does not rely on external servers or actual network connections.

5. تفاصيل التنفيذ
- تم تنفيذ الواجهة باستخدام مكونات JavaFX مثل ListView وTextField وButton.
- تحتوي الشيفرة على منطق لإضافة وحذف المهام ومزامنتها مع معالجة بسيطة للحالات.

5. Implementation Details
- The interface is implemented using JavaFX components such as ListView, TextField, and Button.
- Code includes logic to add, delete, and sync tasks with straightforward handling of user actions.

6. التعليمات التشغيلية
- لتشغيل التطبيق، قم ببناء المشروع باستخدام JDK وتشغيل الفئة الأساسية App.
- استخدم واجهة المستخدم لإدارة المهام عبر القائمتين والأزرار المتاحة.

6. Operational Instructions
- To run the application, build the project using JDK and run the main class App.
- Use the UI to manage tasks across the two lists and interact with the available buttons.

7. التحديات والمشكلات
- الحاجة لمعالجة إدخال فارغ أو غير صالح.
- الحفاظ على تجربة مستخدم سلسة وتحديث القوائم بشكل فوري.
- محاكاة التوزيع ضمن تطبيق مستقل.

7. Challenges and Issues
- Handling empty or invalid input.
- Maintaining smooth user experience and immediate list updates.
- Simulating distribution within a standalone application.

8. التحسينات المحتملة
- إضافة دعم لقوائم مهام متعددة أو مزامنة ثنائية الاتجاه.
- حفظ حالة القوائم بين جلسات الاستخدام.
- تحسين تصميم واجهة المستخدم وتفاعلها.

8. Potential Improvements
- Adding support for multiple task lists or bidirectional synchronization.
- Persisting list state between sessions.
- Enhancing user interface design and interaction.

9. الشاشات والنتائج
- الشاشة الرئيسية تعرض قائمتين للمهام، مربع إدخال، وأزرار التحكم للإضافة، الحذف، والمزامنة.

9. Screens and Results
- The main screen displays two task lists, an input box, and control buttons to add, delete, and sync tasks.

الخاتمة:
يقدم المشروع نموذجًا بسيطًا لفهم إدارة المهام والتوزيع باستخدام JavaFX، مع تركيز على تجربة مستخدم واضحة وبسيطة.

Conclusion:
The project presents a simple model for understanding task management and distribution using JavaFX, focusing on a clear and straightforward user experience.
